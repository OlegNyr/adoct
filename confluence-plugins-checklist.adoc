Все ссылки и совместимость проверены по Marketplace; указаны версии с поддержкой *Confluence Data Center 10.x*. == Примечания

* *Совместимость:* значения DC-версий — на момент проверки; всегда сверяй на странице плагина под свой билд Confluence.
* *Безопасность (важно для банка):* для on-prem предпочитай серверный рендер без egress (PlantUML DC, Scroll PDF, draw.io — хранение на инстансе). У Swagger-плагинов отключай внешний онлайн-валидатор; «Try it out» уводит запросы на реальный API.
* *AsciiDoc / Confluence Publisher:* макросы этих плагинов в редакторе — нативные, но из AsciiDoc автоматически они не появятся. Каждый имеет свой storage-format `ac:name` (напр. draw.io, `swagger`, `pdf-export`) — вставляй их тем же passthrough-приёмом (`++++` + `<ac:structured-macro>`), что и встроенные.
* *Экспорт Confluence→AsciiDoc (наш конвертер):* `drawio` и `plantuml` — выделенные хендлеры. `swagger`/`open-api` — спека OpenAPI выносится в отдельный файл `files/swagger_N.{json,yaml}`, в документ ставится ссылка на него. Для всех прочих плагинных макросов действует generic-правило: макрос с телом (`ac:rich-text-body`) — тело разворачивается (контент Table Filter/AURA/Mosaic не теряется); макрос без тела (динамический листинг) — пропускается. Специфический рендер (Swagger UI, графики, PDF-шаблоны) статически не воспроизводится — остаётся исходный контент/код/спека.
* *Лицензии:* почти всё платное; на DC тарификация идёт по тиру пользователей всего инстанса.
